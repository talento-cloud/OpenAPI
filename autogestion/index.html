Manejo de errores

Los errores habituales pueden incluir:

- `400 Bad Request`: Parámetros requeridos faltantes o con formato
inválido.
    
- `401 Unauthorized`: Token JWT inválido, expirado o ausente.
    
- `403 Forbidden`: El usuario autenticado no tiene permisos suficientes.
    
- `404 Not Found`: El recurso solicitado no existe.
    
- `500 Internal Server Error`: Error inesperado en el servidor.
    

Cuando sea posible, cada endpoint detallará sus códigos de error
específicos y estructura de respuesta en su propia descripción.

solicitudAusentismo

Descripción

Crea registro de solicitud de ausentismo en ag_solicitudes_ausentismo para el usuario suministrado del tipo suministrado.

Las fechas se esperan en formato yyyy-MM-dd y las horas en formato militar

La url pre-firmada que se entrega por cada archivo a cargar en la respuesta es por ejemplo: https://talento-archivos-desarrollo.s3.amazonaws.com/archivos/talento_demo/transversal/x/y/1/z-e8cb-405b-b40d-8ab14da6358f_1776265021877U_1_N_1897_segundo.pdf?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Date=20260415T145701Z&X-Amz-SignedHeaders=host&X-Amz-Credential=AKIAUEA3SQNBJOBO5WOX%2F20260415%2Fus-east-1%2Fs3%2Faws4_request&X-Amz-Expires=900&X-Amz-Signature=9f916967e57d892e696419e6560b7d3c19ad98feac2b401cef31d0acd7b2b55d

Authorizations:
bearer
Request Body schema: application/json
required
usuario
required
integer <int32>

Código del usuario

tipoAusentismo
required
integer <int32>

Código del tipo de ausentismo

fechaInicio
required
string

Fecha inicio en formato yyyy-MM-dd

fechaFin
string

Fecha fin en formato yyyy-MM-dd

horaInicial
string

Hora inicio en formato kk:mm:ss

horaFinal
string

Hora fin en formato kk:mm:ss

observaciones
required
string

Observaciones de la solicitud de ausentismo

filesNames
Array of strings

Opcionalmente se puede adjuntar archivos a la solicitud de ausentismo, para lo cual se manda primero una lista de nombres de estos lo cual luego generará una url pre-firmada para hacer put con el archivo real
