Documentación de los endpoints para la gestión de la Estructura Organizacional Jerárquica.

Flujo de uso

  1. La UI puede consultar los catálogos de Niveles Organizacionales (/niveles-organizacionales) y Cargos (/cargos-activos) para alimentar selectores.
  2. Para visualizar el organigrama, se consume GET /ver-estructura.
  3. La creación de la estructura comienza con POST /primera-posicion para establecer el nodo raíz.
  4. Se añaden nodos hijos con POST /{posicionPadreId}/subordinadas.
  5. Las posiciones vacantes se pueden llenar automáticamente con POST /{posicionId}/asignar-ocupante.
  6. Se puede consultar quiénes son los usuarios sin posición asignada con GET /personas-sin-posicion.

Manejo de errores

Código Descripción
400 Parámetros faltantes o con formato inválido (Bad Request).
401 Token JWT inválido, expirado o ausente (Unauthorized).
403 El usuario autenticado no tiene permisos suficientes (Forbidden).
404 El recurso solicitado no existe (Not Found).
500 Error inesperado en el servidor (Internal Server Error).

Asigna un ocupante a una posición vacante automáticamente.

Busca un candidato adecuado y lo asigna a la posición especificada. La lógica de búsqueda es:

  1. Si la posición padre tiene un ocupante (líder), busca un usuario sin posición que reporte a ese líder y tenga el cargo requerido.
  2. Si no, busca cualquier usuario sin posición que tenga el cargo requerido. Falla si no se encuentra un candidato que cumpla los criterios.
Authorizations:
bearerAuth
path Parameters
posicionId
required
integer <int64>

ID de la posición vacante a la que se le asignará un ocupante.
