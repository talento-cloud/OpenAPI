API Seleccion (1.0)

Download OpenAPI specification:

Contiene servicios web con endpoints pensados para uso de cliente externo a talento, para consultar procesos aprobados en el módulo de selección y tambien para consultar el detalle de un proceso, para poder usarlos debe primero autenticarse con el client_id y client_secret que talento cloud les suministrará

Manejo de errores

Los errores habituales pueden incluir:

- `400 Bad Request`: Parámetros requeridos faltantes o con formato
inválido.
    
- `401 Unauthorized`: Token JWT inválido, expirado o ausente.
    
- `403 Forbidden`: El usuario autenticado no tiene permisos suficientes.
    
- `404 Not Found`: El recurso solicitado no existe.
    
- `500 Internal Server Error`: Error inesperado en el servidor.
    

Cuando sea posible, cada endpoint detallará sus códigos de error
específicos y estructura de respuesta en su propia descripción.

Selección

oauth2

Para generar JWT que se usará en las siguientes peticiones, este tiene un TTL de 15 minutos.

Ejemplo petición con curl:

curl -X POST https://clientex.talento.cloud/api/html/oauth2/token/landers.json \
  -H "Content-Type: application/x-www-form-urlencoded" \          
  -d "grant_type=client_credentials" \
  -d "client_id=mi-id" \
  -d "client_secret=mi-secreto" \
  -d "scope=talento.seleccion.read"

Ejemplo uso token generado:

curl -X POST https://clientex.talento.cloud/api/v1/servicex  \
  -H "Authorization: Bearer eyJhbGci..."
