Documentación del endpoint de listar requisiciones en curso por responsable.

Manejo de errores

Los errores habituales pueden incluir:

- `400 Bad Request`: Parámetros requeridos faltantes o con formato
inválido.
    
- `401 Unauthorized`: Token JWT inválido, expirado o ausente.
    
- `403 Forbidden`: El usuario autenticado no tiene permisos suficientes.
    
- `404 Not Found`: El recurso solicitado no existe.
    
- `500 Internal Server Error`: Error inesperado en el servidor.
    

Cuando sea posible, cada endpoint detallará sus códigos de error
específicos y estructura de respuesta en su propia descripción.

Lista las requisiciones en curso por responsable

Obtiene un listado paginado de las requisiciones (procesos de selección) que se encuentran en curso y están asignadas al usuario administrador o responsable actual. Permite filtrar por estado, filtros de búsqueda abiertos y validación de participación de candidatos.

Authorizations:
bearerAuth
query Parameters
soloTerminados
boolean
Default: false

Indica si se deben listar solo las requisiciones terminadas.

filtrosAdicionales
string

Arreglo JSON en formato string con filtros adicionales para la búsqueda.

codCandidato
integer <int64>

Si se envía, indica si el candidato especificado ya se encuentra en el proceso.

start
integer
Default: 0

Índice inicial para la paginación.

limit
integer
Default: 50

Cantidad máxima de registros a retornar (límite de paginación).

query
string

Cadena de texto libre para búsqueda en los procesos (ej. nombre del cargo, código, etc).
