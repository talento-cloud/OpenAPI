Documentación de los endpoints del módulo de Selección (Requisiciones y Procesos).

Flujo de uso del panel de filtros

Antes de llamar al endpoint principal, la UI ejecuta en paralelo:

  1. GET /seleccion/propiedades/estadoCamposOcultablesRequisicion para determinar qué campos y filtros deben mostrarse u ocultarse.

  2. Los endpoints de catálogo: /sedesPaginado, /ConsultaEmpresasSpring/{negocio}, /cargosold/getPagin, /ConsultaUsuariosSpring/{negocio}/{incluirCorporativo}, /seleccion/plantillaconfiguracion/etapas-proceso/listar/etapas/{codigoPlantilla}/{listarEtapasSeleccionadas}, y /seleccion/aprobacion/flujosAprobacionAll.

  3. Con las selecciones del usuario se construye el parámetro filtrosAdicionales y se invoca GET /requisicion/listarRequisicionesEnCursoPorResposable.

Manejo de errores

Código Descripción
400 Parámetros faltantes o con formato inválido.
401 Token JWT inválido, expirado o ausente.
403 El usuario autenticado no tiene permisos suficientes.
404 El recurso solicitado no existe.
500 Error inesperado en el servidor.

Requisiciones

Operaciones sobre procesos de selección (requisiciones).

Lista las requisiciones en curso por responsable

Obtiene un listado paginado de las requisiciones (procesos de selección) que se encuentran en curso y están asignadas al usuario administrador o responsable actual.

El comportamiento varía según el rol del usuario autenticado:

  • ROL_ADMIN_SELECCION: lista todas las requisiciones en curso del sistema.
  • ROL_ADMINISTRADOR_REGIONAL: lista las requisiciones de su regional.
  • Responsable / Psicólogo: lista únicamente las asignadas a él.

Parámetro filtrosAdicionales

Es un arreglo JSON serializado como string. Cada elemento tiene la forma:

{ "tipo": "<TipoFiltro>", "codigoReal": <valor> }

Tipos válidos (filtros del mismo tipo actúan como OR; entre tipos distintos actúan como AND):

tipo Descripción Tipo de codigoReal Endpoint de catálogo
Sede Filtra por código de sede integer (Long) /sedesPaginado
Empresa Filtra por código de empresa integer (Long) /ConsultaEmpresasSpring/{negocio}
Cliente Filtra por empresa solicitante (cliente externo) integer (Long) /ConsultaEmpresasSpring/{negocio}
Etapa Filtra por etapa actual del proceso integer (Long) /seleccion/plantillaconfiguracion/etapas-proceso/listar/etapas/{codigoPlantilla}/{listarEtapasSeleccionadas}
Cargo Filtra por código de cargo integer /cargosold/getPagin
Tipo Filtra por regla de aprobación (tipo de requisición) integer (Long) /seleccion/aprobacion/flujosAprobacionAll
Solicitante Filtra por usuario solicitante integer (Long) /ConsultaUsuariosSpring/{negocio}/{incluirCorporativo}

Ejemplo completo:

[{"tipo":"Sede","codigoReal":5},
 {"tipo":"Empresa","codigoReal":1},
 {"tipo":"Etapa","codigoReal":12},
 {"tipo":"Etapa","codigoReal":15},
 {"tipo":"Cargo","codigoReal":"520"},
 {"tipo":"Cargo","codigoReal":"522"},
 {"tipo":"Tipo","codigoReal":18},
 {"tipo":"Solicitante","codigoReal":73}]
Authorizations:
bearerAuth
query Parameters
soloTerminados
boolean
Default: false

Si true, retorna únicamente las requisiciones en estado terminado/cerrado. Si false (por defecto), retorna las que están en curso y publicadas.

query
string
Example: query=rss

Búsqueda de texto libre por número de requisición, nombre de cargo o cargo externo.

filtrosAdicionales
string
Example: filtrosAdicionales=[{"tipo":"Sede","codigoReal":5},{"tipo":"Empresa","codigoReal":1},{"tipo":"Etapa","codigoReal":12}]

Arreglo JSON serializado con los filtros adicionales. Ver tabla de tipos en la descripción.

codCandidato
integer <int64>

Si se envía, indica si ese candidato ya participa en cada proceso listado.

start
integer
Default: 0

Índice inicial para la paginación (offset).

limit
integer
Default: 50

Cantidad máxima de registros a retornar.

Lista las empresas activas (para filtros Empresa y Cliente)

Retorna las empresas activas. Si se especifica un negocio, filtra las empresas de ese negocio; de lo contrario retorna todas las empresas activas del sistema.

Se usa para alimentar dos filtros distintos:

  • Filtro Empresa: empresa interna a la que pertenece la requisición.
  • Filtro Cliente: empresa solicitante (cliente externo). Solo visible si lo habilita /seleccion/propiedades/estadoCamposOcultablesRequisicion.

Usar negocio = -1 para obtener todas las empresas sin importar el negocio.

Lista los usuarios del sistema (para filtro Solicitante)

Retorna el listado paginado de usuarios activos. Alimenta el combo "Solicitante" del panel.

El resultado se limita automáticamente según el rol del usuario autenticado:

  • Si no es administrador y no tiene PERMISO_SELECCION_ADMIN, solo ve su cadena de mando.
  • Si se envía moreThanCadenaMando=true, se ignora esa restricción.

El código del usuario seleccionado se envía como {"tipo":"Solicitante","codigoReal":<código>} dentro de filtrosAdicionales.

Authorizations:
bearerAuth

required
string
Example: -1

Código del negocio. Enviar -1 para no filtrar por negocio.

incluirCorporativo
required
string
Example: -1

Parámetro heredado (deprecated). Enviar -1.

query Parameters

Example: modulo=SELECCION

Filtra usuarios que tienen licencia del módulo especificado (ej. SELECCION).

boolean

Lista las etapas de proceso disponibles (para filtro Etapa)

Retorna el catálogo de etapas de proceso. Alimenta el combo "Etapa" del panel de filtros.

Con codigoPlantilla = -1 y listarEtapasSeleccionadas = false se obtiene el listado completo de etapas del sistema, independiente de cualquier plantilla.

Los parámetros mostrarPreselecion y mostrarCierre controlan si esas etapas aparecen en la respuesta, permitiendo mostrar u ocultar esos elementos en la UI.

El código de etapa seleccionada se envía como {"tipo":"Etapa","codigoReal":<código>} dentro de filtrosAdicionales.

Si true, solo incluye etapas ya seleccionadas en la plantilla. Si false, lista todas las disponibles.

boolean
Default: false
Example: mostrarPreselecion=true

Controla la visibilidad de la etapa de Preselección en la UI. Si false, esa etapa no aparecerá en el combo de filtros.

boolean

Controla la visibilidad de la etapa de Cierre en la UI. Si false, esa etapa no aparecerá en el combo de filtros.

integer

limit
integer

Configuración UI

Endpoints que controlan la visibilidad de campos y pestañas en la interfaz de creación/edición de requisiciones. Deben consultarse antes de renderizar la UI.

Estado de visibilidad de los campos en Requisiciones

Retorna un mapa con el estado de configuración de visibilidad (visible/oculto) de campos y pestañas que se usan al crear o editar una requisición.

⚠️ Este endpoint debe consultarse antes de renderizar el panel de filtros. Sus valores determinan qué filtros mostrar al usuario.

Lógica de visibilidad aplicada en la UI

1. 🏢 Campo Empresa Solicitante (tagEmpresaSolicitante)

Solo se muestra cuando se cumplen ambas condiciones:

Propiedad en respuesta Condición
pestana_informacion_cliente.visible true
campo_infcli_empresasolicitante.visible true 
Si (pestaña cliente visible == true)
   Y (campo empresa solicitante visible == true)
→ Mostrar filtro "Empresa Solicitante"
En caso contrario → Ocultar

2. 💼 Campo Cargo (tagCargo)

Se oculta cuando el cargo externo está habilitado:

Propiedad en respuesta Condición
req_cargo_externo.visible true 
Si (cargo externo visible == true) → Ocultar campo "Cargo"
Si no                              → Mostrar campo  "Cargo"
Authorizations:
bearerAuth
