API publica de talento cloud (1.0.0)

Download OpenAPI specification:

API REST para entregar info a datalake. Maneja autenticación Oauth 2.0 y se podria consultar los siguientes informes en formato application/json:

  • Selección - Reporte de selección detallado
  • Selección - Seguimiento por etapas
  • Formación - Avance por tema
  • Bienestar - reporte detallado beneficios/reconocimientos
  • Bienestar - reporte transacciones puntos beneficios/reconocimientos
  • HojaVida - Reporte datos básicos usuario interno
  • HojaVida - Información de nómina
  • Objetivos Smart - Resultados consolidados de gestión (Todo el grupo)
  • Objetivos Smart - Modelo mapa de desempeño
  • Competencias - Detalle valoración de competencias (de todo el grupo)
  • Operativos (Dec) - Reporte operativos
  • Transversal - Reporte de todos los usuarios del sistema
  • Transversal - Reporte de los usuarios activos
  • Transversal - Novedades

Manejo de errores

Los errores habituales pueden incluir:

- `400 Bad Request`: Parámetros requeridos faltantes o con formato
inválido.
    
- `401 Unauthorized`: Token JWT inválido, expirado o ausente.
    
- `403 Forbidden`: El usuario autenticado no tiene permisos suficientes.
    
- `404 Not Found`: El recurso solicitado no existe.
    
- `500 Internal Server Error`: Error inesperado en el servidor.
    

Cuando sea posible, cada endpoint detallará sus códigos de error
específicos y estructura de respuesta en su propia descripción.

NOTAS

Es una propuesta general según lo solicitado en el correo, puede cambiar y le faltan detalles, por ejemplo como indicarle codigos o años de los procesos a consultar, algunos ejemplos de respuesta etc

OAuth2 - Obtener token

Ejemplo petición con curl:

curl -X POST https://clientex.talento.cloud/api/html/oauth2/token/somostalentoalianza.json \
  -H "Content-Type: application/x-www-form-urlencoded" \          
  -d "grant_type=client_credentials" \
  -d "client_id=webhook-servicio" \
  -d "client_secret=mi-secreto" \
  -d "scope=talento.read"

Ejemplo uso token generado:

curl -X POST https://clientex.talento.cloud/api/v1/datalake/modulox/reportey  \
  -H "Authorization: Bearer eyJhbGci..."
